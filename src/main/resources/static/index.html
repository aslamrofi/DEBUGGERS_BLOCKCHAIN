CSBC Digital Identity Management Framework

Blockchain-Anchored Self-Sovereign Identity (SSI) Simulator mapped to Oracle 19c

1. Decentralized Identifier (DID) Generation Issuer Flow

How it works: The system generates an asymmetric RSA 2048-bit KeyPair for the user. The Public Key is anchored securely on the blockchain ledger (your CSBC_BLOCKCHAIN_LEDGER Oracle table) tied to a unique URI pointer (DID). The matching Private Key is returned exclusively to the user's secure wallet. The ledger never stores private keys or actual identity records!Define the student's baseline identity profile below. 2. Cryptographic Verification Portal Verifier Flow

How it works (Decentralized Trust):
  • The user sends a claims statement (JSON payload) along with their unique cryptographic signature.
  • The Verifier queries the Oracle database using the provided DID URI to resolve the corresponding Public Key.
  • The crypto engine processes the data: if even one character inside the identity JSON changes, the verification immediately fails, proving malicious tampering.
