Blockchain-Anchored Self-Sovereign Identity (SSI) Simulator mapped to Oracle 19c

1. Decentralized Identifier (DID) Generation Issuer Flow

How it works (Cryptographic Registration):

This module simulates a Trusted Issuer (University Administration) initializing a student's Decentralized Identity:

  • Generates an asymmetric RSA 2048-bit KeyPair for the student.
  • Anchors the Public Key into Oracle, mapping it to a unique DID URI pointer.
  • Signs student attributes, returning a tamper-proof signature to the holder.

2. Cryptographic Verification Portal Verifier Flow

How it works (Decentralized Trust):
  • The user enters a DID URI, a 4-digit ID, and a cryptographic signature.
  • Verifier resolves the Public Key from Oracle via the DID URI.
  • If the 4-digit ID changes by even one digit, verification fails.
