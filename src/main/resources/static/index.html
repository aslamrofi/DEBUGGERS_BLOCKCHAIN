CSBC Digital Identity Management Framework

Blockchain-Anchored Self-Sovereign Identity (SSI) Simulator mapped to Oracle 19c

1. Decentralized Identifier (DID) Generation Issuer Flow

How it works (Cryptographic Registration):

This module simulates a Trusted Issuer (University Administration) initializing a student's Decentralized Identity:

  • Generates an asymmetric RSA 2048-bit KeyPair for the student.
  • Anchors the Public Key securely into the CSBC_BLOCKCHAIN_LEDGER Oracle table, mapping it to a unique DID URI pointer.
  • Signs the custom student profile attributes dynamically, returning a tamper-proof credential baseline signature to the holder.
Authorized DID Target Formats
Valid identifier URI strings mapping active node profiles inside the database registry:
did:csbc:aslam
did:csbc:hairie
did:csbc:ryan
did:csbc:iqmal
did:csbc:iyad

2. Cryptographic Verification Portal Verifier Flow

How it works (Decentralized Trust):
  • The user sends a claims statement (JSON payload) along with their unique cryptographic signature.
  • The Verifier queries the Oracle database using the provided DID URI to resolve the corresponding Public Key.
  • The crypto engine processes the data: if even one character inside the identity JSON changes, the verification immediately fails, proving malicious tampering.
